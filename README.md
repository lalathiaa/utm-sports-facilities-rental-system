# UTM Sports Facilities Rental System

A Laravel + MySQL sports facilities booking system with an AI module for recommendations and predictive analytics.

---

## Project Setup

### 1. PHP / Laravel

```bash
composer install
cp .env.example .env
php artisan key:generate
php artisan migrate --seed
npm install && npm run dev
php artisan serve
```

### 2. Python Analytics Engine (optional but recommended)

The predictive analytics page (`/analytics`) runs its computations via a Python script
(`scripts/analytics_engine.py`) using **pandas**. This is the primary computation path.

**If Python / pandas is not installed, the page falls back automatically to a pure-PHP
implementation producing identical results. The demo will never break or show a blank
page without Python — the fallback is always available.**

#### Install pandas

```bash
# Install Python 3 (if not already installed): https://python.org/downloads/
pip install -r scripts/requirements.txt
```

#### Configure the Python executable

By default the service calls `python analytics_engine.py`. If your system uses
`python3` (common on Linux/macOS), add this to `.env`:

```dotenv
PYTHON_BIN=python3
```

On **Windows**, `python` is typically correct (Python 3 is registered as `python` via the
Python Launcher). No change needed in most cases.

#### Verify

```bash
echo '{"bookings":[],"facilities":[],"today":"2026-01-01","lookback_days":null,"projection_weeks":4,"underuse_threshold":0.3,"slots_per_day":14}' | python scripts/analytics_engine.py
```

Should output a JSON object with empty arrays — no errors.

### 3. Optional: Google Gemini API (AI recommendation explanations)

The recommendation reasons are generated by the Google Gemini API (free tier).
Without a key, the system generates template-based explanations from the Layer 1
numbers — the feature never breaks.

1. Get a free key at <https://aistudio.google.com/app/apikey>
2. Add to `.env`:

```dotenv
GEMINI_API_KEY=your_key_here
```
